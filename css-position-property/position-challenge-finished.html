The CSS Position Property

The CSS layout algorithms, by default, size and position boxes in relation to each other so that nothing overlaps.

This specification defines several ways to violate these assumptions when needed, moving elements around in ways that can make them overlap other content:

  • Relative positioning, which visually shifts a box relative to its laid-out location.
  • Sticky positioning, which visually shifts a box relative to its laid-out location in order to keep it visible when a scrollable ancestor would otherwise scroll it out of sight.
  • Absolute positioning, which ignores normal layout entirely, pulling the element out of flow and positioning it relative to its containing block with no regard for other content.
  • Fixed positioning, which absolutely positions the box and affixes it to the viewport or page frame so that it is always visible.

These positioning schemes, controlled by the position property and the inset properties, are powerful but easy to misuse. With appropriate care, they allow many interesting and useful layouts that couldn’t otherwise be achieved with standard layout rules; without, they allow a page to be laid out in an unusable overlapping jumble of content. 1

Position: static

The order an element appears in the source code determines where it appears on the rendered page in the browser, resulting in what is known as “normal flow” in the web browser.

Static is the default position value for all elements, unless it is overwritten by the developer.

Position: relative

Relatively positioned elements are positioned the same as statically positioned elements, unless the top, bottom, left or right properties are added.

Elements positioned with the additional properties (top, bottom, left, right) leave the normal document flow, but the space they take up in the document isn’t collapsed. The space stays open and in the same spot the element originally occupied (prior to applying the left, right, top, bottom properties), regardless of where the element moves via the additional CSS properties. Thus, the surrounding elements will not be adjusted to fit into any gap left by the relatively positioned element.

Position: absolute

Absolutely positioned elements are positioned relative to the nearest positioned ancestor using the top, bottom, left and right properties. If the element has no ancestors the body element is used to determine the relative position.

An absolutely positioned element leaves the normal document flow and sit on top of the normal document. The space that would have been occupied by the absolute positioned element in the normal document flow is collapsed. Absolutely positioned elements will scroll with the page content when the page is scrolled.

Position: fixed

A element with a position of fixed is positioned relative to the viewport using the top, bottom, left and right properties.

A fixed positioned element leaves the normal document flow and sit on top of the normal document. The space that would have been occupied by the absolute positioned element in the normal document flow is collapsed. Fixed positioned elements will not scroll with the page content when the page is scrolled.

Position: sticky

A element with a position of sticky combines relative and fixed values to create an element that is positioned relatively until it crosses a threshold where it becomes fixed.
